# sam-os

Personal operating system — schedule, gym, and nutrition tracking. Self-hostable
FastAPI service backed by SQLite, with daily backup to a remote PostgreSQL.

## What it does

- **Schedule** — weekly template + per-day instance tracking (moves, skips, reasons)
- **Gym** — workout log + PR tracking (Epley 1RM)
- **Meals** — calorie/macro logging with daily target adherence
- **Telegram integration** — reactive commands for all three domains (via the `schedule-commands` and `nutrition-commands` skills)
- **OpenAPI docs** at `/docs` (auto-generated by FastAPI)
- **Daily backup** to Neon Postgres (3am local time)

## Quick start (Docker)

```bash
git clone https://github.com/SwayamJoshi87/sam-os.git
cd sam-os
cp .env.example .env
# Edit .env with your DB path and BACKUP_PG_DSN

docker compose -f docker/docker-compose.yml --env-file .env build
docker compose -f docker/docker-compose.yml --env-file .env up -d

curl http://localhost:8765/health
# {"status":"ok","db":"/data/schedule.db"}

open http://localhost:8765/docs   # interactive OpenAPI explorer
```

## Architecture

SQLite lives on the host at the path you specify in `.env`. The container
bind-mounts it read-only, so container restarts never lose data. A separate
backup container syncs the SQLite to Neon Postgres daily.

```
┌────────────────────────────────────────────┐
│  Host                                     │
│  /home/server/data/schedule.db  ← SQLite  │
│  /home/server/sam-os/           ← git     │
└────────────────┬───────────────────────────┘
                 │ bind mount
                 ▼
┌────────────────────────────────────────────┐
│  Docker                                    │
│  ┌────────────────────┐  ┌──────────────┐  │
│  │  sam-os-api        │  │ sam-os-backup│  │
│  │  FastAPI :8765     │  │ APScheduler  │  │
│  │  /api/schedule     │  │ 3am daily    │  │
│  │  /api/gym          │  │ sqlite → pg  │  │
│  │  /api/meals        │  │              │  │
│  │  /docs (OpenAPI)   │  │              │  │
│  └────────────────────┘  └──────┬───────┘  │
└──────────────────────────────────┼──────────┘
                                   │ TLS
                                   ▼
                        ┌─────────────────────┐
                        │  Neon Postgres      │
                        └─────────────────────┘
```

## API reference

See [docs/api.md](docs/api.md) for endpoint documentation, or hit `/docs` after
starting the service for the interactive OpenAPI explorer.

## Endpoints (summary)

### `/api/schedule/*`
- `GET /today` — today's living schedule (instances, not template)
- `GET /week` — full template week
- `POST /did/{task_name}` — mark a task done
- `POST /skip/{task_name}?reason=...` — mark a task skipped
- `POST /push/{task_name}/{day}?permanent=false` — move a task (one-off or permanent)
- `GET /history?days=7` — completion log
- `GET /stats?days=7` — completion stats by category

### `/api/gym/*`
- `POST /log` — log a workout (with auto-PR detection)
- `GET /prs` — list all PRs (filter by `?gym=office`)
- `GET /recent?days=7` — recent workouts

### `/api/meals/*`
- `POST /log` — log a meal
- `POST /target` — set today's calorie/macro target
- `GET /today` — today's meals + totals vs target
- `GET /week` — last 7 days adherence
